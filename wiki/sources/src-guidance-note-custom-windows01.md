# NatHERS Guidance Note 1 — Custom Windows (WERS Value Discrepancies)

## Summary

One-page NatHERS Guidance Note 1 (effective 27 July 2017, amended August 2017, for software using Chenath engine V3.13) setting the tolerance for discrepancies between custom window values in NatHERS software libraries and the corresponding Window Energy Rating Scheme (WERS) data. It deems a custom window the same for compliance purposes when the stated U-value and/or SHGC differ from WERS data by less than 10% and the manufacturer, window type, frame type and glazing type all match.

## Key claims

- Where a custom window selected in NatHERS accredited software shows a discrepancy in stated U-value and/or Solar Heat Gain Coefficient (SHGC) against the corresponding WERS data of **less than 10%**, the window is considered the same for compliance purposes — provided ALL of the following match: window manufacturer, window type (e.g. awning, sliding, casement), frame type (e.g. timber, aluminium), and glazing type (e.g. clear, single glazed, double glazed low-e).
- Variations falling outside these bounds should be raised directly with the Australian Fenestration Rating Council (AFRC) and copied to the NatHERS Administrator (admin@nathers.gov.au).
- Effective from 27 July 2017, for use with software using Chenath engine V3.13; amended August 2017.

## Concepts touched

NatHERS, glazing, windows, U-value, SHGC, WERS, AFRC, custom windows, Chenath engine

## Caveats

- Dated 2017 and tied to Chenath V3.13-era software. Current window substitution rules are in the NatHERS Technical Note Version 20241023 (identical opening type, Uw equal or greater, SHGCw ±5%, AFRC-based values) — see src-nathers-technical-note-20241023.md — and the WERSlink libraries arriving with Chenath 3.23. Verify this 10% tolerance is still accepted before relying on it for a current assessment.
